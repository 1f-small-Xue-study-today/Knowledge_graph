# Memory: Build Environment

## Tools and Platforms

- **MCU:** HC32F460 (Cortex‑M4), built with C compiler (Keil or GCC).
- **Simulation:** MATLAB/Simulink for algorithm design and rapid prototyping.
- **Visualisation:** Python (e.g., Matplotlib) for plotting waveforms and analysing data.
- **Version Control:** Use Git to manage source code and Beads issues.
- **Issue Management:** Beads for task graph and persistent memory.  Memory entries such as this one should be stored as Beads issues with tag `memory`.

## Notes

- Ensure that the toolchain versions are documented to avoid inconsistencies.
- Maintain a consistent directory structure for source code, simulation scripts and documentation.
- Use continuous integration to compile and run unit tests automatically.
